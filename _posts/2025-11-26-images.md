GitHub 블로그(Jekyll)에 이미지를 넣는 가장 정석적인 방법이다.

핵심은 **"이미지를 저장할 폴더를 만들고, 그 경로를 마크다운에 적는 것"**이다.

---

## 1단계: 이미지 폴더 만들기

프로젝트 루트(최상위) 위치에 이미지를 모아둘 폴더를 만든다.  
보통 `assets` 안에 `images`를 만드는 것이 관례다.

폴더 구조 예시는 다음과 같다.

```text
gimini-3.github.io/
├── _posts/
│   └── 2025-11-26-start.md
├── assets/          <-- 이 폴더 생성
│   └── images/      <-- 이 안에 생성
│       └── test.png <-- 여기에 사진 넣기
├── _config.yml
└── index.md
```
## 2단계: 이미지 파일 이름 규칙

이미지 파일 이름은 반드시 영문 소문자와 숫자로 짓는 것이 좋다.

좋은 예:
- aws-architecture.png
- spring-boot-1.jpg

피해야 할 예:
- 스크린샷 2025.png

- 내 사진 최종 최종.png

한글, 공백, 특수문자가 섞여 있으면 \
배포 시 경로 인코딩 문제 때문에 이미지가 깨질 확률이 높다.

## 3단계: 마크다운에 이미지 넣기

글 파일(예: _posts/2025-11-26-start.md)에서 아래 문법을 사용한다.
```
![이미지 설명](/assets/images/파일명.png)
```

실제 예시는 다음과 같다.
```
# AWS 구조도

아래는 제가 설계한 아키텍처입니다.

![AWS Architecture](/assets/images/aws-architecture.png)
```

이렇게 적으면 /assets/images/aws-architecture.png 경로에 있는 이미지가 글에 표시된다.

---
## 주의할 점: 경로 설정

경로 맨 앞에 **슬래시(/)** 를 꼭 붙여야 한다.

assets/images/... ❌
→ 현재 글이 있는 폴더(_posts) 기준 상대 경로라,
Jekyll 빌드/배포 시 이미지를 제대로 찾지 못할 수 있다.

/assets/images/... ✅
→ 사이트 루트(/) 기준 절대 경로라,
어느 글에서든 안정적으로 이미지를 불러올 수 있다.

---
## 마무리

1. assets/images 폴더를 만들고

2. 영어/숫자 이름으로 이미지를 넣은 뒤

3. 글에서 ![설명](/assets/images/파일명.png) 형식으로 경로를 적고

4. git add, git commit, git push 하면

GitHub Pages(Jekyll) 블로그에서 이미지를 정상적으로 볼 수 있다.
