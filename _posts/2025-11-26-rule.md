오늘은 GitHub Pages를 이용해서 기술 블로그를 만들었다.


GitHub Pages(Jekyll 엔진)가 인식하는 필수 규칙들을 정리한다.

## 1. 글 파일 이름 규칙 (엄격함)
_posts 폴더 안의 파일은 반드시 아래 형식을 따라야 한다.

- 형식: YYYY-MM-DD-영문제목.md

- 필수: 연, 월, 일, 하이픈(-), 제목이 모두 있어야 한다.

- 주의:

  - 2025-11-26.md (X) → 제목이 없어서 인식 불가. 
  - hello-world.md (X) → 날짜가 없어서 인식 불가. 
  - 2025-11-26-start.md (O) → 정상.

## 2. 디렉토리(폴더) 명명 규칙
Jekyll은 폴더 이름 앞의 언더바(_) 유무에 따라 처리 방식이 다르다.

- _posts: 글 파일이 저장되는 곳. 언더바(_)와 복수형(s) 필수다.

- _config.yml: 환경 설정 파일. 루트 경로에 있어야 한다.

- assets 또는 images: 이미지나 CSS 파일 등을 넣는 곳. 여기는 언더바를 붙이지 않는다. (언더바가 붙은 폴더의 파일은 웹에서 직접 주소로 접근할 수 없다.)

## 3. Front Matter (머리말) 규칙
모든 글 파일(md)의 맨 첫 줄은 반드시 Front Matter로 시작해야 한다.
- 형식: --- 로 감싸야 한다.
- 위치: 파일의 1번째 줄이어야 한다. 그 앞에 공백이나 빈 줄이 있으면 안 된다.

```Markdown
---
layout: post
title: "GitHub Pages로 기술 블로그 시작하기"
date: 2025-11-26 10:00:00 +0900
categories: blog start
---
(여기서부터 본문)
```


## 4. 날짜 및 시간 규칙
- 미래의 글: 설정된 date가 현재 시간(GitHub 서버 기준 UTC)보다 미래일 경우, 해당 글은 숨김 처리되어 보이지 않는다.
- 해결: 시간을 넉넉하게 과거로 잡거나, _config.yml에 future: true 설정을 추가해야 한다.

## 5. 변경 사항 반영 규칙
   - /수정: git push 후 약 1~3분 내에 자동 반영된다. 
   - _config.yml 수정: 설정 파일을 건드렸다면, 변경 사항이 크므로 빌드 시간이 조금 더 걸릴 수 있다. 반영이 안 되면 웹 브라우저 캐시를 삭제하고 새로고침 해봐야 한다.
